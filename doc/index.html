FilterLogger

The FilterLogger class is a special class of logger that passes a filter on a message before logging it. As such, the FilterLogger class does no logging. Instead, it forwards the responsibility to log to another logger after having modified the message.

The FilterLogger class takes in parameter any number of filters. The filters must extend the LogFilterInterface interface.

Here are sample use cases:

  • You could use a filter to discard a special log message that is repeated too often and useless.
  • You could use a filter to automatically add meta data like categories to a log before sorting it into database (using the DB_Logger).
  • etc...

By default, the FilterLogger package comes with one preconfigured filter: EnhanceCategoryLogFilter. This filter takes the log message, keeps the first 30 characters (configurable), and use those characters as a category for the log. If you forward this to the DB_Logger, it will automatically classify the log using the category.
